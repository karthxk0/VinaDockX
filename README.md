# VinaDockX v2.12
A multipurpose python tool to make AutoDock Vina based docking and virtual screening simple and time saving 

## Usage
- Download and extract `VinaDockX.zip` from latest [Releases](https://github.com/karthxk0/VinaDockX/releases) or [Click here](https://github.com/karthxk0/VinaDockX/archive/refs/heads/main.zip)
- Open `Terminal` in the extracted directory
- Run : `python VinaDockX.py`


## Prerequisites & Installation

### 1. Minimum Requirements
* **Python:** Version 3.8 or higher.
* **Dependencies:** The script is designed to be plug-and-play. It will **automatically detect and install** any missing Python packages (such as `MDAnalysis`, `numpy`, `biopython`, `rdkit`, `meeko`, `gemmi`, `pandas`, `openpyxl`, `colorama`, and `tqdm`). 

### 2. Downloading the Tool
- Download and extract `VinaDockX.zip` from latest [Releases](https://github.com/karthxk0/VinaDockX/releases) or [Click here](https://github.com/karthxk0/VinaDockX/archive/refs/heads/main.zip)
- Alternatively, clone the repository directly from GitHub using your terminal or command prompt:

```bash
# Clone the repository
git clone https://github.com/karthxk0/VinaDockX.git
```

## Running VinaDockX

- Navigate into the extracted/cloned directory
  
```bash
cd VinaDockX
```

- Run the script from terminal using python:

```bash
python VinaDockX.py
```

## Functioning

- Upon launching, the main menu offers two primary modes of operation:
  1.  **Complete Docking Pipeline:** Runs sequentially from Step 1 through Step 7.
  2.  **Partial Pipeline:** Allows you to select specific modules (e.g., typing `1, 3, 5-7` to skip       grid generation and protein prep).

```text
Main Menu/
├── 1. Complete Docking Pipeline (Scripts 1-7)
└── 2. Run Pipeline Partially
    ├── 1 InterResFi: Find Interacting Residues/
    ├── 2 GridConfigGen: Generate Grid & Config/
    ├── 3 GridViz: Visualize grid/
    ├── 4 PrepProt: Protein preparation/
    ├── 5 PrepLig: Ligand Preparation/
    ├── 6 VinaDock: Docking Execution/
    └── 7 BindReSort: Sort docking results/
```


## Core Modules 

### Step 1: `InterResFi` (Find Interacting Residues)
* **Task:** Analyzes a co-crystallized protein-ligand complex (PDB) to automatically detect and extract amino acid residues within a 6Å radius of the ligand.
* **Use Case:** Crucial for setting up targeted grid boxes or determining which residues to make flexible during docking.

### Step 2: `GridConfigGen` (Generate Grid & Config)
* **Task:** Calculates the spatial coordinates (center x, y, z and size x, y, z) for the docking search space and writes them into an AutoDock Vina `.txt` configuration file.
* **Modes:** * *Blind:* Encompasses the entire protein.
    * *Targeted:* Focuses only on specific interacting residues (either auto-detected from Step 1 or provided manually by you).

### Step 3: `GridViz` (Visualize Grid)
* **Task:** Generates visual representations of the defined grid box over the receptor. 
* **Use Case:** Allows you to inspect and confirm that the search space is correctly positioned before spending computational resources on docking.

### Step 4: `PrepProt` (Protein Preparation)
* **Task:** Converts standard `.pdb` receptor files into the `.pdbqt` format required by Vina. It handles the addition of polar hydrogens and Gasteiger charges.
* **Modes:** Standard (Rigid) or Flexible docking. If Flexible is chosen, it will split the receptor into rigid and flexible `.pdbqt` components based on your residue inputs.

### Step 5: `PrepLig` (Ligand Preparation)
* **Task:** Batch converts ligand files (from `.sdf` or `.mol2` formats) into `.pdbqt` files suitable for virtual screening. Supports processing entire folders of ligands at once.

### Step 6: `VinaDock` (Docking Execution)
* **Task:** The core engine. It orchestrates AutoDock Vina, feeding it the prepared receptor(s), ligands, and configuration file. It will systematically dock every ligand provided in the input directory.

### Step 7: `BindReSort` (Sort Docking Results)
* **Task:** Scans the messy text log files generated by AutoDock Vina, extracts the top binding affinities (kcal/mol) for each ligand, and compiles them neatly into a single, easy-to-read `.xlsx` Excel spreadsheet.

## Step-by-Step Usage Guide

VinaDockX uses an interactive prompt system. You do not need to memorize complex command-line flags. Just answer the prompts on the screen:

1.  **Select Output:** You will first be asked where you want to save the results. Provide a base directory path.
2.  **Choose Search Space (If running Step 2):** * Choose between *Blind* or *Targeted*.
    * If *Targeted*, you can either let the tool auto-detect residues from a complex or type them manually (e.g., `A:12, B:15`).
3.  **Choose Docking Type (If running Step 4):**
    * Choose between *Rigid* or *Flexible*.
    * If *Flexible*, define which residues should be allowed to move.
4.  **Provide File Paths:** Depending on the steps chosen, simply drag-and-drop or type the paths to your Receptor (`.pdb`), Ligands, or Config files when prompted.
5.  **Confirm & Run:** The tool will print a summary of your configuration. Type `y` to confirm, and the pipeline will execute while displaying a progress bar.

## Understanding the Outputs

VinaDockX keeps your workspace highly organized. Every time you run a pipeline, it creates a unique, timestamped master folder in your chosen output directory:

**Example Output Structure:**
```text
Your_Output_Folder/
└── TargetProtein_VinaDockX_29032026-1430/
    ├── TargetProtein_VinaDockX-Log_29032026-1430.txt  <-- Master Log of the entire run
    ├── 1 Interacting Residues - InterResFi/           <-- .txt files of extracted residues
    ├── 2 Grid & Config - GridConfigGen/               <-- Vina config.txt files
    ├── 3 Grid Visualizer - GridViz/                   <-- Grid visualization files
    ├── 4 Protein Preparation - PrepProt/              <-- Receptor .pdbqt files (rigid/flex)
    ├── 5 Ligand Preparation - PrepLig/                <-- Ligand .pdbqt files
    ├── 6 Docking Results - VinaDock/                  <-- Raw Vina outputs and logs
    └── 7 Sorted Results - BindReSort/                 <-- The final Excel (.xlsx) summary
```

* **Intelligent Logging:** The master log file (`...VinaDockX-Log...txt`) records every configuration choice, terminal output, and error, making it incredibly easy to troubleshoot or reproduce your work later.
* **Cascading Failures:** If a crucial step fails (e.g., a receptor fails to prepare in Step 4), VinaDockX is smart enough to safely skip Step 6 rather than crashing, and it will note this in the final summary.
